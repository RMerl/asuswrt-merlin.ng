Broadcom MDK Programmer's Reference

2. Architecture Overview

The Mini Driver Kit (MDK) is divided into three separate components, which are the Chip Development Kit (CDK) and the Broadcom Mini Driver (BMD) and the PHY Driver Library (PDL).

The CDK provides a full description of all chip blocks, registers and memories and includes a low-level interface for reading and writing registers and memories. The CDK has no abstractions, i.e. all register and memory access is chip-specific. The CDK also includes a simple debug shell that provides symbolic or raw access to the chip registers.

The BMD is a simple API built on top of the CDK, and it optionally adds corresponding shell commands to the CDK debug shell.

The PHY Driver Library is actually a separate component, which can be used in other applications not involving the BMD, however for any real BMD applications, one or more PHY drivers must be included.

In addition to the MDK components, an MDK-based application must also provide a basic system interface for hardware registers access and optionally console I/O.
