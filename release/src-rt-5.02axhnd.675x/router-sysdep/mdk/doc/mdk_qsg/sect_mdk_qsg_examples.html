Broadcom MDK Quick Start Guide

3. Example Applications

The following sections will explain how to build a simple MDK application in a Linux/Unix environment.

3.1. Simple CDK Shell

This section will explain how to build a simple debug shell based on the CDK. The sample application does not require a target system since all register access is emulated.

At this point you have probably already installed the MDK package. Now you must set the CDK environment variable to point to the CDK package, e.g. from the MDK root directory (bash syntax): 

export CDK=`pwd`/cdk
Now go to the sample application directory and run GNU make: 
cd examples/cdksim
gmake
Assuming that the build completes without errors, go ahead and run the application: 
./cdksim
Hopefully you are now seeing the CDK prompt: 
CDK.0>
The zero in the prompt is the current unit number. To get information about this unit, type unit at the prompt (followed by ENTER).

At any time you can type help to get a list of available commands. If you need detailed help on a command, simply type help command-name, e.g. help help.
